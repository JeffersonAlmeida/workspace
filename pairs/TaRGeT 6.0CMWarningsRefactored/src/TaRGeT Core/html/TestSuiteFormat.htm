Test Suite Format

The generated Test Suite format follows the TestCentral format. A test case is composed by:

  • The Use Cases field, which lists all use cases related to the test case.
  • The Requirements field, which is filled with all requirements covered by the test case.
  • The Setup field, which is filled with the setup information from the use cases (see the use case structure).
  • The Initial Conditions field, which is filled with all system states covered by the test case.
  • The Procedure field, which is a sequence of user actions from the use cases (see the use case structure).
  • The Expected Results field, which is the sequence of system responses for its corresponding actions (see the use case structure)

As you can see, there are other test case fields in the example above. However, the content of those fields is not generated, or are generated with a default text.
