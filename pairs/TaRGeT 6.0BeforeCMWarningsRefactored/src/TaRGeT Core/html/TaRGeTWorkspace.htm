TaRGeT Workspace

TaRGeT Workspace displays three different panels (Figure 1):

                                                                                        Figure 1 - Target Workspace

 

  • Left Panel

This panel (Figure 2) contains the Use Cases view in which the use cases are outlined. A tree structure is shown, grouping the use cases by feature. The Features folder contains all the use cases grouped according to their feature.

             Figure 2 - Left Panel

 

  • Bottom Panel

This panel allows three views:

  • Artifacts View – it outlines the project artifacts. It groups, in two different folders, the use case documents and the test suite documents (Figure 3).

                                                Figure 3 - Artifacts View

 

  • Errors View – it lists all errors in the use case documents. Each error is composed by the Description, the Resource where the error ocurred, the Path of the resource and the error Location inside the resource (Figure 4). 

                                                    Figure 4 - Errors View

 

  • Search View – this list displays the results of a search operation. It lists all use cases that are found according to the given query. Each result is composed by the Use Case Name, the Use Case Id, the id of the feature that contains the use case (Feature ID) and the name of the document that contains the use case (Document Name) (Figure 5).

                                                    Figure 5 - Search View

 

  • Main Panel

This panel is used to display the use case documents content (Figure 6) and the On The Fly Generation editor (Figure 7).

 

                                                                            Figure 6 - Use Case View

 

                                                                    Figure 7 - On The Fly Generation editor
