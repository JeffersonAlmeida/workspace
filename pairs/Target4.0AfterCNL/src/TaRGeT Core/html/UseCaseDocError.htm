Errors and Warnings in Documents

In TaRGeT, the document defects are classified in errors and warnings . If the project contains any type of error, TaRGeT will not generate tests. On the other hand, TaRGeT generates tests from projects that contain warnings. 

There are two kinds of errors in TaRGeT. They are listed below:.

  • Simple error - The simple errors do not compromise the processing of the document. They are:
    • Duplicated Step Id Error - Occurs when more than one step, inside the same use case, presents the same ID.
    • Empty Field Error - Occurs when the step id, action or response are empty. If a use case has more than one error, only one will be displayed in the Errors view.
    • Invalid Step Id Reference Error - Occurs when the document (in the To Steps and/or From Steps fields) has an invalid step reference.
    • Invalid To/From Steps Error -  Occurs when To/From Steps are not in the Step ID format.
  • Fatal error - The fatal errors compromise the processing of the document, so its contents will not be displayed in the TaRGeT Workspace. They are:
    • Unable To Load Document Error - Occurs when the document is not a Microsoft Word, or it is Microsoft Word but does not have attached XML, or it is Microsoft Word but the XML is not according to the pre-defined template.
    • Duplicate Use Case Id Error - Occurs when there is more than one use case, inside the same Feature, with the same ID.
    • Empty Use Case Id Error - Occurs when one use case ID is empty.
    • Invalid Use Case ID Error - Occurs when the Use Case ID field is composed by symbols or "numbers_letters", e.g. "01_UC".
    • Duplicate Feature Id Inside Document Error - Occurs when there is more than one feature, inside the same document, with equal IDs.
    • Empty Feature Id Inside Document Error - Occurs when the feature ID is empty.
    • Invalid Feature ID Error - Occurs when there are any content different from numbers in the Feature ID.
    • Empty To/From Steps Error - Occurs when From Step ot To Step fields are empty.
    • Invalid Step ID Error - Occurs when it is not in the Step ID format (Numbers + Uppercase Letters)

Regarding the warnings, TaRGeT consider the follow ones:

  • Features With Same Id But Different Names - TaRGeT accepts two or more features with same Id that are placed in different documents. Actually, TaRGeT considers the features as only one. Although the features have the same Id, they may have different names, which sounds mistaken in the TaRGeT environment. So, the user is notified with a warning message.
