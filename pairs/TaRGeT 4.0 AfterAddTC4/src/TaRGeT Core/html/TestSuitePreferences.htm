Test Suite Preferences

 

 TaRGeT allows some customization on test suite fields. For this, before generation the test suite, go to Tools > Preferences  from the menu bar.

 

In this screen you can configure your preferences related to some options around the contain that shall displayed in the test suite:

 

  • Test Case ID - This property sets the test case identification field. The property may be composed of optional parameters: the <tc_featureid> parameter, which is filled with the feature ID, and the <tc_id> parameter, which is an ID automatically generated by TaRGeT;
  • Test Case Initial ID - This property sets the initial id value of the test suite;
  • Empty Field - This property sets the fields that remain empty in the generated test cases;
  • Objective Prefix - This property sets the prefix content of the Objective field;
  • Print Use Case Description - This property sets how the Use Case description shall be composed in the test suite. This description the Use Case in the test suite can be composed by All descriptions of the use cases or by Last description of the use case or yet by None description of the use case.
  • Print Flow Description - This property sets how the Flow description shall be composed in the test suite. This description the Flow in the test suite can be composed by All descriptions of the flows or by Last description of the flow or yet by None description of the flow.
  • Keep Requirements - This property is a option that, if selected, shall show the requirements in the step of the test case in addition to showing it in the requirements field of the test case
